## 상황

React Query + Next.js App Router에서 `HydrationBoundary`로 SSR prefetch를 걸었어요. 그런데 HTML에 데이터가 하나도 안 실리고, 서버 로그도 안 찍히더라고요. `prefetchQuery`는 분명 호출되는데 말이죠.

## 원인

코드를 보니 이렇게 돼 있었어요.

```typescript
// use-get-series-bundles.ts
'use client'; // ← 이게 문제

export const fetchSeriesBundlesServer = async (country) => {
  const res = await fetch(url);
  // ...
};

export const useGetSeriesBundles = () => useQuery({ /* ... */ });
```

```typescript
// prefetch-series-bundles.ts
import { fetchSeriesBundlesServer } from './use-get-series-bundles'; // ← 'use client' 파일에서 import

export async function prefetchSeriesBundles(queryClient, country) {
  await queryClient.prefetchQuery({
    queryFn: () => fetchSeriesBundlesServer(country), // 서버에서 실행 안 됨
  });
}
```

Next.js는 `'use client'` 가 붙은 파일 **전체**를 Client Reference Module로 처리해요. 서버 번들엔 실제 코드가 안 들어가고 클라이언트 참조(reference)만 남습니다. 그래서 서버에서 그 함수를 호출해도 아무 일도 안 일어나요. `prefetchQuery` 의 `queryFn` 이 실행 자체가 안 된 거예요.

결과적으로 이렇게 흘러갔어요.

- `prefetchQuery` 호출은 성공하지만 `queryFn` 은 실행 안 됨
- `dehydrate(queryClient)` 에 데이터가 없음 → HTML에도 데이터 없음
- 클라이언트에서 다시 fetch → 깜빡임 발생, SSR한 의미가 사라짐

## 해결

서버에서 돌아야 하는 함수는 `'use client'` 파일과 분리하면 됩니다. prefetch 파일 안에서 직접 정의하는 게 제일 명확했어요.

```typescript
// prefetch-series-bundles.ts (수정 후) — 'use client' 파일 import 없음
const fetchSeriesBundlesServer = async (country) => { // 파일 내부에서 직접 정의
  const res = await fetch(url);
  // ...
};

export async function prefetchSeriesBundles(queryClient, country) {
  await queryClient.prefetchQuery({
    queryFn: () => fetchSeriesBundlesServer(country), // 정상 실행
  });
}
```

## 남길 판단

함수 종류에 따라 자리를 나누는 규칙으로 정리했어요.

| 함수 종류 | 위치 |
|---|---|
| React hook (`useQuery` 등) | `'use client'` 파일 |
| 서버 전용 fetch 함수 | `'use client'` 없는 파일 (prefetch 파일 내부 등) |

핵심은 이거예요. **파일에 `'use client'` 가 붙는 순간, 그 파일의 모든 코드는 서버에서 실행 불가능하다고 가정해야 합니다.** 훅이랑 서버 fetch 함수를 한 파일에 편하게 몰아넣다가 딱 이 함정에 빠졌어요. prefetch가 조용히 안 먹으면 제일 먼저 import 출처에 `'use client'` 가 묻어 있는지부터 보세요.
