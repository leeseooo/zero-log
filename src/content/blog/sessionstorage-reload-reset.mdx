챗봇 열림 상태를 세션스토리지에 저장하고 있었는데, **새로고침하면 초기화하고 싶은데 첫 진입이나 상태 복원이랑 구분하기가 애매했어요.** 처음엔 `beforeunload` 로 붙잡아보려다 자꾸 새로고침인지 창 닫기인지 헷갈리는 문제에 부딪혔습니다.

## 해결책: `PerformanceNavigationTiming`

마운트 시점에 "이 진입이 새로고침이었나?"를 물어보면 됐어요.

```typescript
useEffect(() => {
  const navEntry = performance.getEntriesByType(
    'navigation'
  )[0] as PerformanceNavigationTiming;

  if (navEntry?.type === 'reload') {
    // 새로고침이면 초기화
    sessionStorage.setItem(KEY_IS_CHAT_BOT_OPEN, 'false');
    setIsOpen(false);
  } else {
    // 첫 진입 or 이전 상태 복원
    setIsOpen(getSessionStorageItem(KEY_IS_CHAT_BOT_OPEN) === 'true');
  }
}, []);
```

cleanup 함수랑 `beforeunload` 는 다 걷어내도 됐어요.

**왜 동작하냐면요,**
- 새로고침 = 전체 페이지 재로드 → layout 포함 컴포넌트가 새로 마운트 → `useEffect` 재실행
- 그 마운트 시점에 `navEntry.type === 'reload'` 로 새로고침 여부를 딱 판단

## beforeunload vs PerformanceNavigationTiming

둘을 비교하면 이렇게 갈려요.

| | beforeunload | PerformanceNavigationTiming |
|---|---|---|
| 시점 | 페이지가 파괴되기 직전 | 페이지가 새로 로드된 직후 |
| 정확도 | 새로고침인지 닫기인지 구분이 모호 | `type: 'reload'` 로 명확히 구분 |
| 상태 | 휘발성 (이벤트 내부에서 비동기 처리 어려움) | 안정적 (마운트 시점에 확인 가능) |

**언제 뭘 쓰냐면요,**
- **PerformanceNavigationTiming** — 새로고침으로 온 건지 파악해서 상태를 초기화하거나 초기 UI를 결정할 때
- **beforeunload** — 브라우저를 잘못 닫았을 때 alert를 띄우거나, 로그아웃 API를 호출해야 할 때

## 주의점

아주 오래된 브라우저에서는 deprecated된 `performance.navigation.type` 을 써야 할 수도 있어요. 지금 코드의 `getEntriesByType('navigation')` 이 표준이니, 특별한 레거시 지원이 필요한 게 아니면 이걸로 가면 됩니다.
